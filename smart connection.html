Hieronder wordt per week de content geplaatst voor het thema Smart Connection.

Smart Connection WK1

Tijdens de eerste week van het thema Smart Connection was er de gelegenheid om de ontwikkelomgevingen in te richten. Zo werd er gezorgd dat iedereen toegang kreeg tot de platformen Moodle en GitHub. Deze platformen zullen worden gebruikt om lesmateriaal te verkrijgen én opdrachten in te leveren.

Voor mij, als student van Hogeschool Saxion, was dit een erg nuttige bijeenkomst. Ik was nog niet zo goed bekend met de systemen van de HAN, maar tijdens de workshop was er voldoende gelegenheid om hier uitleg over te krijgen. Na de uitleg kon ik gelijk een stuk beter overweg met de systemen van de HAN.

Zo rond week 4 had ik mijn wegpagina zo ingericht dat ik hem kon voorzien van content, die ik gedurende de eerste paar weken van de minor had verzameld. Verder heb ik een account aangemaakt op de website https://openweathermap.org/, waarna ik een API Key heb ontvangen.

Na het kijken van de filmpjes had ik gelijk een veel beter beeld van wat een API inhield. Een Application Programming Interface (API) is naar mijn inzicht een "hulpmiddel om data op te halen vanuit een server". Deze data wordt teruggestuurd naar de "client" (die de aanvraag heeft gedaan) en kan velerlei van format zijn bijv. XML, JSON etc. Om het voor mijzelf duidelijker te maken had ik een beknopte schets gemaakt wat een API precies doet. Een foto van deze schets is hieronder weergegeven:

Tijdens de zeer interactieve workshop in week 5 gingen wij zelf, met behulp van JavaScript, informatie over de weersomstandigheden ophalen uit de OpenWeatherMap's API. Dit hebben we gedaan door via een stappenplan van OpenWeatherMap een HTML-pagina op te bouwen. Zo werd er allerlei JavaScript content toegevoegd met de bijbehorende CSS opmaak. Op de website van OpenWeatherMap was tevens een lijst beschikbaar met City ID's. Via deze lijst konden we de ID van onze eigen woonplaats opzoeken, in mijn geval de plaats Lichtenvoorde.

De werking van een dergelijke API is als volgt:

Een Client, ik dit geval mijzelf, vraagt door middel van een HTTP-request naar de temperatuur én luchtdruk in de plaats Lichtenvoorde. Vervolgens ontvangt de REST API deze aanvraag, de REST-API doet op zijn beurt een beroep op de database om de benodigde gegevens te benaderen (temperatuur + luchtdruk in Lichtenvoorde). Als de REST-API de benodigde gegevens heeft verzameld, dan worden deze in het gewenste format (in dit geval JSON) teruggestuurd naar de Client, ofwel naar mijzelf.
